### 概要
本リポジトリは、Kaggleコンペティション「Predicting Heart Disease」 の分析ををまとめたものです。  
現在のPublic Leaderboard 順位：236位 / 約2663人  
評価指標：ROC-AUC  
使用モデル：XGBoost,Catboost  

### データセット
約63万行の大規模データ  
欠損値なし  
目的変数：Heart Disease（二値分類）  

### 特徴量エンジニアリングの考え方
本コンペでは、以下のような設計判断を行いました。  
1. 交差特徴量の検証  
まず、カテゴリ変数同士や数値特徴量同士のさまざまな交差特徴量を作成し、性能改善を試みました。  
しかし、CVスコアの改善が見られませんでした。  
この結果から、本データセットでは元の特徴量の表現方法を工夫する方が有効であると判断しました。  

2. 数値型派生特徴量の作成  
次に、数値特徴量に対して以下の派生特徴量を追加しました。  
対数変換（log1p） 二乗項（square）  
これらの特徴量により、分布の歪みや非線形な関係を補足でき、モデルの表現力が向上しました。  

3. 数値特徴量をカテゴリ化  
通常、数値のカテゴリ化はビニングすることが多いです。  
しかし、本データセットは約63万行と大規模であり、数値のユニーク数が最小42（Age）〜最大150（Cholesterol)と限定的でした。  
1つのユニーク値あたり平均4,000〜15,000件のサンプルが存在するため、ビニングで情報を落とさずにカテゴリ化した方が精度が良くなったと考察しています。
